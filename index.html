The language that ships
with everything.

Winn is a compiled language on the BEAM with clean Ruby-inspired syntax, a built-in HTTP server, ORM, testing framework, and AI pipelines. No external infrastructure. Just write code and ship.

$ brew install gregwinn/winn/winn

Ruby's readability.
Elixir's power.
Zero infrastructure.

Winn compiles to the BEAM — the same battle-tested virtual machine that powers WhatsApp, Discord, and Ericsson's telecom infrastructure. You get lightweight processes, hot code reloading, fault tolerance, and distributed computing built into the runtime.

But unlike Erlang or Elixir, Winn ships with an HTTP server, an ORM, a testing framework, a documentation generator, and a file watcher with a live dashboard. No setup. No boilerplate. No twenty dependencies to install before you can write a web app.

Everything you need. Nothing you don't.

HTTP Server

Cowboy-powered with route matching, middleware, JSON responses, and path parameters. No Express, no Flask, no Rails — it's built in.

ORM & Migrations

Schema DSL, changesets, validations, connection pooling, transactions. PostgreSQL + SQLite. Query with User.all(), User.find(1).

Testing Framework

winn test with assert, assert_equal, automatic test discovery, colorized output, and exit codes. Write tests in Winn, not Erlang.

Pipe Operator

Chain transformations with |>. Data flows left to right, top to bottom. No nested function calls.

Pattern Matching

Multi-clause functions, destructuring, guards, switch expressions, and match blocks. The compiler picks the right clause for you.

OTP Built In

GenServer, Supervisor, and Application behaviours with use Winn.GenServer. Fault-tolerant by default.

Hot Code Reloading

winn watch shows a live terminal dashboard with per-module status, reload times, and compile errors inline.

Doc Generator

winn docs generates Markdown API docs with a Mermaid dependency graph that renders on GitHub.

Clear Error Messages

Elm/Rust-style compiler errors with source context, caret pointers, and hints. No cryptic stack traces.

Structs & Protocols

Named struct types with struct [:name, :email]. Define interfaces with protocol and implement them with impl.

Rails-Style Generators

winn create model User name:string — scaffold models, migrations, routers, tasks, and full CRUD. winn migrate for schema versioning.

Deploy Anywhere

winn release builds a production OTP release. winn release --docker generates a Dockerfile. The BEAM runs forever.

HTTP Server

module Api
  use Winn.Router

  def routes()
    [{:get, "/users/:id", :get_user}]
  end

  def get_user(conn)
    id = Server.path_param(conn, "id")
    {:ok, user} = User.find(id)
    Server.json(conn, user)
  end
end

Pattern Matching

module Math
  def fib(0)
    0
  end

  def fib(1)
    1
  end

  def fib(n)
    fib(n - 1) + fib(n - 2)
  end
end

Testing

module MathTest
  use Winn.Test

  def test_addition()
    assert(1 + 1 == 2)
  end

  def test_greeting()
    result = "hello" <> " world"
    assert_equal("hello world", result)
  end
end

Models & Database

module User
  use Winn.Schema

  struct [:name, :email, :age]

  schema "users" do
    field :name, :string
    field :email, :string
    field :age, :integer
  end
end

# Rails-style queries
users = User.all()
user  = User.find(1)
{:ok, user} = User.create(%{name: "Alice"})

What's coming next.

Winn isn't just a language. It's becoming a platform.

AI-Native Pipelines

AI.chat(), AI.classify(), AI.extract() as stdlib. Pipe data through AI models like you pipe through Enum. Agent DSL for autonomous tools. Multi-provider support.

Distributed Events

Event.emit(:order_created, order) and on :order_created do |order| ... end. Pub/sub across BEAM nodes. No Kafka. No Redis. Just Winn.

Background Jobs

use Winn.Job with queues, retries, cron schedules, and a live dashboard. Database-backed. No Sidekiq. No external worker process.

Built-in Observability

winn bench for load testing. winn metrics for a live metrics dashboard. Auto HTTP metrics, custom counters, BEAM VM health.
